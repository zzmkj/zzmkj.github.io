Oauth2结合JWT
发表于2019-12-12|更新于2019-12-30
|字数总计:1.5k|阅读时长:6分钟|阅读量:

Spring Cloud Security 为构建安全的SpringBoot应用提供了一系列解决方案，结合Oauth2还可以实现更多功能，比如使用JWT令牌存储信息，刷新令牌功能.

1. JWT简介

JWT是JSON WEB TOKEN的缩写，它是基于 RFC 7519 标准定义的一种可以安全传输的的JSON对象，由于使用了数字签名，所以是可信任和安全的。

1.1.JWT的组成

  • JWT token的格式：header.payload.signature；
  • header中用于存放签名的生成算法；
  • payload中用于存放数据，比如过期时间、用户名、用户所拥有的权限等；
  • signature为以header和payload生成的签名，一旦header和payload被篡改，验证将失败。

3.oauth2中存储令牌的方式

之前我们都是把令牌存储在内存中的，这样如果部署多个服务，就会导致无法使用令牌的问题。Spring Cloud Security中有两种存储令牌的方式可用于解决该问题，一种是使用Redis来存储，另一种是使用JWT来存储。
